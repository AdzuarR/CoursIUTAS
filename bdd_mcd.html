Le Modèle Conceptuel de Données

Objectif du MCD

L'objectif du MCD est de faire un schéma représentant les données ainsi que les dépendances fonctionnelle existantes dans la base de donnée

Règles de construction du MCD

Dans un MCD, les Rectangles représentent les entités, ils ont un nom d'entité qui est généralement un nom commum. Et possèdent une ou plusieurs propriétés

Les ovales représentent les associations entre les entités, leur nom est de préférence un verbe. Ils peuvent avoir zéro, une ou plusieurs propriétés

La propriétés dans un MCD est le plus petit élément d'information

La relation dans un MCD est l'association qui relie deux entités, elle peut ne pas avoir de propriétés

Autres règles des MCD

Une entité peut être reliée à elle même par une relation

Une relation peut relier une, deux, ou plus de deux entités entres elles

L'ensemble des donnée du dictionnaire de donnée doit exister en tant que propriétés dans le MCD>

Notion d'Occurence en MCD

Définition

Une occurence est une valeur possible pour l'/la {Entité,propriétés,Relation} dont il est question

  • L'occurence d'une propriétés est une valeur possible pour cette propriétés

  • L'occurence d'une entité est un ensemble ayant une existence propre d'occurence de ses propriétés

  • Aucune propriétés ne peut prendre plus d'une valeur
